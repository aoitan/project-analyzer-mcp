# Phase 3: ディレクトリ・モジュールマップ（俯瞰）APIの実装

## 概要

ディレクトリ（パッケージ）ごとの「レイヤー境界とImport方向（依存の向き）」を俯瞰し、LLMにトップダウンでドメイン分割を理解させるためのAPIを提供する。これは単なるファイルツリー（`ls -R`）以上の意味合いを持ち、新規機能追加の際の「当たりづけ」や循環参照の警告等に用いられる。

## 要件

- ファイル間のImportステートメントの解析によるディレクトリ単位での依存方向の計算
- `get_module_dependencies(DirPath)` ツールの定義
  - 指定されたパッケージが依存している（Importしている）他のパッケージのリスト
  - 指定されたパッケージに依存されている（Importされている）他のパッケージのリスト
- 「/core はドメイン・ロジックであり、/ui はプレゼンテーション層である」といったメタデータ的ヒントをサーバー側で付与してレスポンスに含める（LLMがレイヤー境界を意識するためのGraph RAGフォーマット構築）

## 関連仕様

- `doc/graph_rag/specification.md` の「Phase 3: ディレクトリ／レイヤー境界の俯瞰」

## 完了条件

- 指定パッケージやディレクトリを起点として、プロジェクト全体でのそのパッケージの立ち位置（依存・被依存関係）が取得できること
- テストが追加されていること
