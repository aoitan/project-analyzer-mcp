# Phase 1: クラス関連グラフ（アーキテクチャ・ディスカバリー）APIの実装

## 概要

LLMにとって最も価値の高い「最初の手がかり（骨格）」となる、クラス図レベルの構造依存関係を動的探索するAPI（ツール）を提供する。指定したクラスから見た親クラス、実装プロトコル、および自身がプロパティとして保持（依存）している他のクラスへのエッジなどを一括で取得できるようにする。

## 要件

- インターフェース、プロトコル、クラス（具象クラス含む）の定義情報とプロパティ群のAST（またはアダプタ経由）からのパース
- 以下のような情報をLLMに返す `get_class_architecture(ClassName)` または類似機能のツールの定義
  - クラス名・シグネチャ・プロトコル
  - 親クラス（Inheritance）のリスト
  - 実装しているインターフェース（Realization）のリスト
  - DIで注入されたプロパティや変数として持つ依存関係クラス（Dependency/Aggregation）のリスト
- これらを組み込んだ「Graph RAG」用JSONレスポンスの策定。単なるリストではなく、人間のエンジニアに向けた「要約メタデータ（自然言語による簡単な補足説明）」をレスポンスに含める（LLMの「Lost in the middle」の防止策）。
